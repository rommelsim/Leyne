Audited the live `design-remake` branch UI (`ios-native/Leyne/WhereSia/`, root swapped in via `WSRoot` — see [[wheresia_redesign]]) specifically against modern iOS design language (Liquid Glass, Dynamic Type, native components), not general usability. Overall grade: **Mixed** — strong, disciplined custom design system (typography, restrained motion, colour-as-data rule) but architecturally opts OUT of iOS 26 Liquid Glass entirely, so on the platform-native axis it reads as dated/cross-platform-generic despite the polish.

**Why:** deployment target is 18.0 (`IPHONEOS_DEPLOYMENT_TARGET = 18.0` in project.pbxproj) and the app suppresses every native chrome surface that would auto-render Liquid Glass on iOS 26 devices: `WSRoot.swift` hides the nav bar on every push (`.navigationBarBackButtonHidden(true)` + `.toolbar(.hidden, for: .navigationBar)`, ~lines 100-105) and draws its own flat tab bar (`WSTabBar`, ~133-177) and header (`WSHeaderBar`, `WSComponents.swift` 436-459) with solid `Color` fills — zero hits for `Material`/`glassEffect`/`Glass`/`blur(` anywhere in the module (grep-verified). Search is a `.fullScreenCover` with a hand-built TextField+Cancel, not `.searchable()`. Zero `.sheet(`/`presentationDetents` anywhere. Every single `Button` in the module uses `.buttonStyle(.plain)` (grep-verified, no semantic/system button styles adopted at all) with fully custom backgrounds — meaning no default press-state feedback anywhere. Zero haptics (`sensoryFeedback`/`Haptic`/`UIImpactFeedback` all absent, grep-verified).

Specific bugs found (not just style opinions):
- `WSSegmented` and `WSFilterChips` (`WSComponents.swift:381,409`) use raw `.onTapGesture` instead of `Button` — no default VoiceOver button trait, no press state. `WSToggle` (341-361) same pattern at line 357, and hand-reimplements a native `Toggle`+`.tint()` from scratch including manually-added a11y traits.
- Tab bar inactive state (icon + label) uses `ws.faint` on `ws.bg` (`WSRoot.swift:162,170`) — computed contrast ≈3.05:1 dark / ≈2.39:1 light, both fail WCAG AA 4.5:1 for text. Same `ws.faint`-on-dark-surface failure repeats for real content (not just decoration) at `WSBusStopView.swift:91,152,213`, `WSMrtStationView.swift:98`.
- Icon-only tap targets are 38×38 (`WSHairButton`, `WSComponents.swift:417-432`) — below the 44×44pt HIG minimum; search's clear button is 20×20 (`WSSearchView.swift:56-62`).
- `WSTheme.sans()/mono()` (`WSTheme.swift:88-97`) use `Font.custom(name, fixedSize: UIFontMetrics.default.scaledValue(for:))` — the `fixedSize:` initializer is documented to NOT auto-scale with Dynamic Type; they're manually working around that with `UIFontMetrics.default` (ambient `UITraitCollection.current`, not guaranteed live in a pure-SwiftUI host). Apple's documented fix for custom fonts + Dynamic Type is `.custom(_:size:relativeTo:)`. Verify behavior across Dynamic Type / accessibility (AX1-AX5) categories before trusting this pattern.
- `WSIcons.swift:8` doc comment claims single- vs double-deck bus "each get a distinct symbol," but `case .busBendy: return "bus"` (line 27) is identical to `case .busSingle: return "bus"` (line 25) — bendy buses are visually indistinguishable from single-deck buses.

What's genuinely well done (don't regress these): pure SF Symbols via one semantic `WSIcons.swift` wrapper; the "colour = MRT lines only" rule is followed with real discipline (`WSTheme.swift:9-13` documents it, `accent` blue is used only for the explicitly-documented exception list of selected/live states); ALL motion (`WSEntrance`, `CrowdGauge` fill, `WSPing` radar, `ForecastBar` grow) is gated behind `accessibilityReduceMotion`; `CrowdGauge`/`WSPing` are `.accessibilityHidden(true)` and always paired with a spoken word per the file's own stated rule (`ArrivalPill.pillLabel`, `WSComponents.swift:246-250`, is a good compound-VoiceOver-label example); light mode DOES exist and is wired (`WSTheme.light`, `WSMeView` appearance cycle) — it isn't dark-only despite the "departure board" framing suggesting otherwise.

**How to apply:** before any future WhereSia UI work, check this is still current (files may have moved post-audit). Top fix priority if asked to modernize: (1) let the tab bar / nav bar sit on `.ultraThinMaterial`/iOS 26 `glassEffect()` instead of flat `Color` fills, or migrate to real `TabView`+`.toolbar` to get Liquid Glass for free; (2) fix `ws.faint`-on-dark/light contrast, especially the tab bar; (3) bump icon-only touch targets to 44×44; (4) add `.sensoryFeedback` to toggle/bookmark/alert-CTA moments; (5) swap `.onTapGesture` for `Button` in `WSSegmented`/`WSFilterChips`.
